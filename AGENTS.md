# AGENTS.md

This is the **corpus repository** for legalize-pe. It contains only Markdown files (legal norms) and git history. The code that produces these files lives in [`crafter-research/legalize-pe-engine`](https://github.com/crafter-research/legalize-pe-engine).

If you are an AI agent editing files here, read this first.

## What this repo is

- **Data**, not code. Markdown norms with YAML frontmatter following [SPEC v0.2](https://github.com/legalize-dev/legalize/blob/main/SPEC.md).
- Listed in the [legalize.dev federation hub](https://github.com/legalize-dev/legalize) under "Community".
- Public domain content (Decreto Legislativo 822, Art. 9). Repository structure: MIT.

## Layout

```
pe/                              # Nacional: leyes, DLeg, DS, Constitución (1,622 Markdown files)
pe/reformas-constitucionales/    # 31 Leyes de Reforma Constitucional
pe-cus/                          # Cusco regional ordinances
pe-{iso}/                        # 25 ISO 3166-2:PE regional codes + pe-lim, pe-cal (most pending V2)
AUDIT.md                         # Coverage report (generated by engine)
```

## Commit identity

**Every commit to this repo MUST be authored by**:

```
Crafternauta <the.crafter.station@gmail.com>
```

with the **real publication date** of the norm as the git author date and committer date.

If a human edits a file directly (rare), they may commit under their own identity, but the rest of the corpus is bot-authored. Audit trail: `git log --author=Crafternauta` shows pipeline output, `git log --author=<human>` shows hand corrections.

Pre-1970 norms: git rejects author dates before the Unix epoch. Use `1970-01-02T00:00:00Z` for the git date and keep the real date in the frontmatter `publication_date` and the `Source-Date` trailer.

## Commit message format

```
[<type>] <Title>[ arts. <comma-separated>]

Source-Id: <reform-id-or-norm-id>
Source-Date: YYYY-MM-DD
Norm-Id: <norm-id>
```

Types per SPEC v0.2: `bootstrap`, `reform`, `new`, `repeal`, `correction`, `fix-pipeline`.

## Frontmatter

Eight mandatory fields, English keys:

```yaml
---
title: "Constitución Política del Perú"
identifier: "CON-1993"
country: "pe"
rank: "constitucion"
publication_date: "1993-12-30"
last_updated: "2024-12-11"
status: "in_force"
source: "https://lpderecho.pe/constitucion-politica-peru-actualizada/"
---
```

Peru-specific extensions are flat (not nested under `extra:`): `jurisdiction`, `scope`, `issuing_entity`, `official_journal`, `affected_articles[]`, `applied_reforms[]`, `gob_pe_slug`, `pdf_url`, etc.

## How to add a norm (manual, rare case)

Most norms get added by the engine. If you must add one by hand:

1. Check the identifier convention in [`AGENTS.md`](https://github.com/crafter-research/legalize-pe-engine/blob/main/AGENTS.md) of the engine repo.
2. Place the file at `pe/{ID}.md` (national) or `pe-{iso}/{ID}.md` (regional).
3. Use Crafternauta identity, real publication date, SPEC trailers.

Example:

```bash
GIT_AUTHOR_NAME="Crafternauta" \
GIT_AUTHOR_EMAIL="the.crafter.station@gmail.com" \
GIT_AUTHOR_DATE="2024-09-28T00:00:00Z" \
GIT_COMMITTER_NAME="Crafternauta" \
GIT_COMMITTER_EMAIL="the.crafter.station@gmail.com" \
GIT_COMMITTER_DATE="2024-09-28T00:00:00Z" \
  git commit -m "$(cat <<'EOF'
[new] Decreto Legislativo N.° 1668

Source-Id: DLEG-1668-2024
Source-Date: 2024-09-28
Norm-Id: DLEG-1668-2024
EOF
)"
```

## What NOT to do

- Do not run `git filter-branch` or `git filter-repo` without coordinating with [@Railly](https://github.com/Railly). Federation tools depend on commit hashes.
- Do not change file names without renaming via `git mv` and preserving history.
- Do not add code (TypeScript, scripts, etc.) here. Engine code lives in `crafter-research/legalize-pe-engine`.
- Do not add `Co-Authored-By: Claude` or other AI tags.
- Do not use em dashes (`—`) in commit messages or PR descriptions.
